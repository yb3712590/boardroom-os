# 后置与冻结范围

> 说明：这里记录“仓库里还在，但当前不主动扩张”的能力边界，以及已经明确进入远期储备的方向。当前主线仍以 `../roadmap-reset.md` 和 `../TODO.md` 为准。

## 已实现但暂停扩张

下面这些能力保留现状，但默认不继续扩张，除非它们直接服务当前本地 MVP：

- 多租户 worker 运维面与 `worker-admin` HTTP 控制面
- 操作人令牌、可信代理断言、独立 auth rejection 读面
- multipart artifact upload
- 本地默认 + 可选对象存储双后端
- 更强的远程 worker handoff 与公网暴露边界

## 明确后置的方向

在本地 MVP 闭环完成前，不主动推进下面这些方向：

- 新的 `worker-admin` 读写接口
- 更细的多租户 scope 治理与租户级运维工具
- 面向公网的身份层、外网暴露策略、零信任边界
- 浏览器直传、云厂商预签名直传、远程对象存储交付链路增强
- 为远程多节点部署设计的新控制面复杂度
- 新的检索层或向量数据库，除非已经证明当前本地历史摘要不够用

## `R1` 远期储备

下面这些方向已经被明确判定为不进入当前关键路径：

| 方向 | 当前口径 | 为什么后置 |
|------|----------|------------|
| `SPIKE_TICKET` | 重大影响项，不进当前 `TODO` | 会同时碰 ticket 类型、调度、投影和前端识别 |
| 分层上下文的 `L3 -> L2` 受控提升 | 远期储备 | 当前只吸收 `L1` 纪律，`L2` 仍由既有检索承接 |
| 项目地图 / project map 真相源 | 远期储备 | 当前不能新增新的项目真相文件 |
| 组织过程资产体系 | 远期储备 | 依赖更重，不直接缩短本地 MVP 路径 |
| 组织学习闭环 | 重大影响项，远期储备 | 属于元治理能力，当前阶段验证成本过高 |
| 系统化文档治理引擎 | 远期储备 | 当前阶段只按治理规则和文档约束处理 |
| “并发实施、串行收口”的系统化调度能力 | 远期储备 | 当前只作为治理原则记录，不做调度器改造 |
| 交付完成的硬门禁 | 远期储备 | 当前只接受 checker / closeout 的软约束，不做硬状态机 gate |

## 使用方式

- 这些内容不是“马上做”，也不是“彻底删除”
- 如果某轮工作必须碰到这些区域，只允许做最小解堵，不允许顺手扩接口、补平台化抽象或重写目录
- 真正要把这些方向拉回主线时，先更新 `../roadmap-reset.md`、`../TODO.md` 和 `../milestone-timeline.md`
